# Security Policy

## Supported Versions

Only the latest published version is supported for security fixes.

## Reporting a Vulnerability

Please report vulnerabilities privately. Do not open public issues for active exploits.

Include:

- affected version/commit
- impact and attack path
- reproducible steps or proof-of-concept

## Scope

Primary risk areas:

- SQL validation bypasses
- dataset boundary escapes
- authentication misconfiguration
- sensitive data exposure in logs/errors

## Disclosure Process

1. Acknowledge report within 3 business days.
2. Reproduce and assess severity.
3. Release patch and mitigation guidance.
4. Publish disclosure notes after fix availability.
